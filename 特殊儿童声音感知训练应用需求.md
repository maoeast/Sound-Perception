# 特殊儿童声音感知训练应用需求
## 1. 项目概述 (Project Overview)

- **项目目标:** 开发一款面向特殊学校儿童（如自闭症、听障康复、智力发育迟缓）的五大声音感知训练桌面应用。
- **技术栈:** Tauri (Desktop), Rust (Backend), Frontend (请基于你的实际框架填入，如 React/Vue/Svelte), Local File System (本地存储)。
- **核心五大声音类别:** 动物 (Animals)、大自然 (Nature)、交通 (Transport)、乐器 (Instruments)、日常生活 (Daily Life)。

## 2. 全局设计原则与约束 (Global Design Principles & Constraints)

- **极简抗干扰:** UI 必须保持极简，使用大色块和高对比度。禁止使用花哨背景、不必要的装饰元素和复杂的导航菜单。
- **高容错度:** 错误操作不提供刺耳/负面的听觉反馈（如错误音效），改用视觉弱化（变暗或轻微晃动）。正确操作提供强烈的正向反馈。
- **无时间压力:** 任何交互环节**绝对禁止**设置倒计时功能。
- **防误触机制:** 核心训练界面按钮需最大化，并考虑集成 Kiosk Mode (全屏锁定) API，防止意外退出。
- **感官脱敏:** 预留全局“柔和模式”配置项，用于一键限制音频峰值、关闭所有动态闪烁效果。

## 3. 核心功能模块 (Core Feature Modules)

### 3.1 客户端交互模块 (Client Interface)

- **自由探索模式 (认知):** 界面展示各类别素材卡片。点击卡片触发声音和微小正向动效。
- **引导训练模式 (匹配):** 播放目标声音，展示 2~4 个候选项供选择。
- **场景沉浸模式 (进阶):** 展示完整场景图，播放特定声音，要求用户在复杂视觉中找出声音源。

### 3.2 教师管理控制台 (Admin Dashboard)

- **数据看板:** 记录学生训练时长、正确率分布、对特定声音类别的敏感度/抗拒度数据。
- **干预配置:** 支持全局难度调节（选项数量动态设置）。
- **自定义资产:** 利用 Tauri 本地 API，支持教师上传本地图片和录音素材，以实现个性化泛化训练。

## 4. AI 辅助资产生成策略 (AI Asset Generation Strategy)
本项目没有专业美术人员，要求 AI 助手在开发过程中协助生成图片素材，可以利用生图技能apimart-imagegen。

### 4.1 图片生成规则 (Prompt Engineering Rules)
当要求生成图片时，AI 必须遵守以下特殊儿童视觉认知规则：

- **主体风格:** 强制使用“极简矢量 (Minimal Vector)”或“柔和插画 (Soft Illustration)”。
- **背景要求:** 强制使用纯白色或高对比度单色背景，确保主体剥离。
- **光影与细节:** 消除复杂阴影、反光、纹理和无关背景杂物。
- **类别特殊约束:**
  - 动物类: 强调可爱特征，去除野性/惊悚感。
  - 大自然类: 色彩柔和，降低动态范围。
  - 日常生活/交通类: 认知一致性强，具有泛化训练价值。
