# 特殊儿童声音感知训练应用 MVP 设计说明

## 1. 目标与范围

### 1.1 产品目标

构建一款面向特殊学校儿童的离线 Windows 桌面训练应用，帮助儿童在低干扰、高容错的交互环境中完成声音认知和声音匹配训练。

### 1.2 本次范围（MVP）

本次仅实现以下能力：

- `自由探索模式`：点击素材卡片，播放对应声音并展示温和正向反馈。
- `引导训练模式`：播放目标声音，展示 2～4 个候选项，记录是否选择正确。
- `教师管理台`：提供匿名统计看板、训练设置和素材管理。
- `内置默认素材包`：应用首次启动即可直接训练。
- `自定义素材上传`：教师可追加本地图片和音频素材。
- `柔和模式`：限制音量峰值，关闭闪烁类动效。
- `全屏防误触`：训练界面优先全屏，降低误退出概率。
- `隐藏教师入口 + 4 位教师口令`：防止儿童误入管理区。

### 1.3 明确不做

以下能力不进入本次 MVP：

- `场景沉浸模式`
- `学生档案`
- `云同步`
- `联网依赖`
- `运行时 AI 生图`
- `强锁定式系统级 Kiosk`

## 2. 关键约束

- 平台仅支持 `Windows`。
- 应用必须支持 `完全离线` 运行。
- 不允许使用负向声音反馈。
- 不允许引入倒计时或时间压力。
- 首版统计基于 `匿名使用`，不区分具体学生。

## 3. 技术方案

### 3.1 技术栈

- 桌面壳：`Tauri 2`
- 前端：`React + TypeScript + Vite`
- 状态管理：`Zustand`
- 本地数据库：`sql.js`（按用户要求采用 `sqlite.js` 路线）
- 本地文件能力：`@tauri-apps/plugin-fs`
- 文件选择器：`@tauri-apps/plugin-dialog`
- 音频引擎：`Web Audio API`
- 单元测试：`Vitest + React Testing Library`

### 3.2 为什么采用 `sql.js`

本项目数据量小、单机单窗口、无高并发写入需求，使用 `sql.js` 足以覆盖 MVP。

采用该方案的核心原因：

- 数据模型仍然保留 `SQLite` 的结构化能力，后续迁移成本较低。
- 数据量较小，内存数据库 + 导出文件的持久化方式可接受。
- 训练记录与配置写入频率不高，性能不是瓶颈。
- 省去 Rust 侧数据库封装，首版前后端边界更简单。

同时明确接受以下代价：

- 数据库默认在内存中运行，需要显式导出二进制文件。
- 不适合多窗口同时写入。
- 不适合后续演化为复杂同步或大数据量分析场景。

这里的实现口径统一为：**前端使用 `sql.js` 运行 SQLite，数据库文件以二进制快照方式持久化到 Tauri 的 `AppLocalData` 目录。**

## 4. 总体架构

应用采用 `Tauri 单体桌面应用` 架构，分为 4 个内部模块：

1. `训练端 UI`
2. `教师管理台`
3. `音频与反馈引擎`
4. `本地存储与素材管理`

### 4.1 模块关系

- 训练端 UI 调用音频引擎播放声音并触发反馈。
- 训练端 UI 将训练行为写入本地数据库。
- 教师管理台从本地数据库读取统计并修改全局设置。
- 素材管理模块负责复制教师上传的文件，并写入素材元数据。

## 5. 信息架构

### 5.1 儿童侧页面

儿童可见页面只保留 3 个：

- `主页`
- `自由探索`
- `引导训练`

主页仅展示两个超大按钮：

- `自由探索`
- `引导训练`

页面不出现多层导航，不出现文本密集说明，不出现复杂状态栏。

### 5.2 教师侧页面

教师台包含 3 个页面：

- `数据看板`
- `训练设置`
- `素材管理`

教师入口不在常规导航中展示，采用隐藏触发方式：

- 默认方案：右下角热区连续点击 5 次
- 触发后弹出 `4 位 PIN` 输入层

## 6. 数据设计

### 6.1 表结构

#### `settings`

用于存储全局配置：

- `id`
- `teacher_pin_hash`
- `soft_mode_enabled`
- `guided_option_count`
- `fullscreen_enabled`
- `first_run_completed`
- `created_at`
- `updated_at`

#### `assets`

用于存储图片和音频素材元数据：

- `id`
- `category`：`animals | nature | transport | instruments | daily_life`
- `title`
- `image_path`
- `audio_path`
- `source_type`：`builtin | custom`
- `enabled`
- `sort_order`
- `created_at`
- `updated_at`

#### `sessions`

用于记录每一次训练会话：

- `id`
- `mode`：`explore | guided`
- `category`
- `started_at`
- `ended_at`
- `duration_ms`

#### `attempts`

用于记录每一道引导训练题目：

- `id`
- `session_id`
- `target_asset_id`
- `candidate_asset_ids_json`
- `selected_asset_id`
- `is_correct`
- `answered_at`

#### `interaction_events`

用于记录行为事件，支撑“敏感度/抗拒度”近似分析：

- `id`
- `session_id`
- `asset_id`
- `event_type`：`play | replay | skip | complete | early_exit`
- `event_at`

### 6.2 统计口径

MVP 阶段不做人脸、情绪、复杂抗拒算法，仅做轻量行为代理指标：

- `训练时长`
- `按模式统计正确率`
- `按类别统计正确率`
- `按素材统计播放次数`
- `按类别统计跳过次数`
- `按类别统计提前退出次数`

可视化文案应使用「可能偏好」「可能抗拒」这类弱结论，不给出过度确定的诊断性描述。

## 7. 存储策略

### 7.1 数据库持久化

- 应用启动时：
  - 检查 `AppLocalData/app.db` 是否存在。
  - 若不存在，则创建空数据库并初始化表结构。
- 应用运行时：
  - 所有查询和写入在 `sql.js` 内存数据库中完成。
- 持久化时：
  - 在配置、素材、训练记录发生变化后，触发 `debounce 1～2 秒` 的数据库导出。
  - 将 `db.export()` 生成的 `Uint8Array` 写入 `AppLocalData/app.db`。

### 7.2 素材文件存储

- 默认素材存放在应用资源目录中。
- 首次启动时，将默认素材复制到 `AppLocalData/assets/builtin/`。
- 教师导入素材时，将原文件复制到：
  - `AppLocalData/assets/custom/images/`
  - `AppLocalData/assets/custom/audio/`

数据库中只保存相对路径，不保存绝对路径，降低迁移和打包风险。

## 8. 训练模式设计

### 8.1 自由探索模式

自由探索模式用于建立「图片 - 声音」的基本关联。

交互规则：

- 每屏展示单一类别的大卡片网格。
- 点击卡片后立即播放声音。
- 正向反馈为卡片放大、描边高亮或轻微发光。
- 无错误概念，因此不展示失败反馈。

### 8.2 引导训练模式

引导训练模式用于建立声音匹配能力。

交互规则：

- 随机选择一个目标素材播放声音。
- 展示 `2～4` 个候选卡片，数量由教师设置决定。
- 选对时展示强烈正反馈，并记录一次正确。
- 选错时只做弱化视觉反馈，不播放惩罚音。
- 支持重新播放目标声音。
- 不设置倒计时。

## 9. 音频与反馈设计

### 9.1 音频播放要求

- 统一通过 `Web Audio API` 播放，避免多个音频实现混用。
- 所有音频先走总线控制，便于实现柔和模式。
- 允许对短音效做预加载，减少点击后延迟。

### 9.2 柔和模式

柔和模式启用后：

- 降低总音量上限。
- 对瞬时峰值做压缩。
- 关闭闪烁、快速缩放和强刺激动效。
- 保留必要的正向反馈，但改为平缓节奏。

### 9.3 错误反馈

错误反馈仅允许：

- 轻微变暗
- 轻微晃动
- 短暂禁用按钮后恢复

错误反馈明确禁止：

- 刺耳音效
- 红色警报闪烁
- 倒计时催促

## 10. 教师管理台设计

### 10.1 数据看板

展示匿名整体数据：

- 总训练次数
- 总训练时长
- 自由探索播放次数
- 引导训练正确率
- 各类别正确率
- 各类别跳过 / 提前退出趋势

### 10.2 训练设置

MVP 中仅开放以下设置：

- `柔和模式` 开关
- `引导训练候选项数量`（2 / 3 / 4）
- `训练界面默认全屏` 开关
- `教师 PIN` 修改

### 10.3 素材管理

教师可以：

- 查看全部素材
- 启用 / 禁用素材
- 上传新图片和音频
- 为素材指定类别和标题

MVP 不做批量标签、波形编辑、裁剪和在线转码。

## 11. Windows 桌面行为

- 应用首版仅面向 `Windows` 打包。
- 主训练窗口默认全屏或准全屏展示。
- 关闭按钮弱化或隐藏。
- 退出到教师台、关闭应用等高风险操作必须经过教师入口。

这里不追求系统级强锁定式 Kiosk，而是优先实现：

- `儿童难以误触退出`
- `教师可控退出`
- `课堂内可快速恢复`

## 12. AI 素材生成策略

AI 生成图片仅用于开发期或备课期准备默认素材，不放入运行时产品流程。

生成约束沿用需求文档：

- 风格优先 `极简矢量` 或 `柔和插画`
- 白底或高对比纯色底
- 去除复杂纹理、阴影和背景噪声
- 动物类强调可爱、非惊悚

## 13. 风险与缓解

### 13.1 `sql.js` 持久化风险

风险：

- 崩溃前的未导出数据可能丢失。

缓解：

- 对关键写操作启用短延迟自动保存。
- 应用隐藏或退出前强制执行一次保存。

### 13.2 默认素材包质量风险

风险：

- 首版素材数量不足，会削弱可用性。

缓解：

- 每个类别至少提供一组可直接训练的图片 + 音频。

### 13.3 Windows 音频设备兼容风险

风险：

- 个别设备首次播放延迟较高。

缓解：

- 启动时预热音频上下文，常用音频预加载。

## 14. 首版验收标准

- 安装后无需联网、无需导入素材即可开始训练。
- `自由探索模式` 和 `引导训练模式` 均可用。
- 教师可以进入后台查看匿名统计、修改设置并导入素材。
- 柔和模式能明显降低刺激性反馈。
- 儿童无法直接看到教师入口或轻易误退出训练。
