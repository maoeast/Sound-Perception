# 特殊儿童声音训练页面布局调整设计说明

## 1. 目标与范围

### 1.1 本轮目标

在不破坏现有 `44` 项正式内置素材显示与播放能力的前提下，调整儿童侧主要训练页面的布局，使其更适合 `27` 寸一体机触摸屏使用。

本轮重点解决的问题：

- 当前主容器过窄，大屏下页面像“缩在中间的一小块”。
- 首页主入口不够突出，训练开始动作不够有重量。
- 自由探索与引导训练的首屏层级偏弱，触摸目标偏保守。
- 大屏可视面积没有被有效利用，但又不能做成信息密集的后台界面。

### 1.2 本轮范围

只调整以下 `3` 个页面：

- `主页`
- `自由探索`
- `引导训练`

### 1.3 明确不做

以下内容不进入本轮：

- 不重做素材接入
- 不重做图片生成
- 不重做音频整理
- 不调整 `round2`、`preview-only` 等历史链路
- 不回头排查 `Tauri` 白屏、`localhost refused`、环境问题
- 不修改教师端页面布局
- 不调整训练逻辑、出题逻辑、音频路径、图片路径和统计逻辑

## 2. 已确认决策

### 2.1 页面范围

用户已确认，本轮只做：

- `首页 + 自由探索 + 引导训练`

### 2.2 视觉方向

用户已确认采用 `C. 平衡型触摸布局`，其含义为：

- 大屏优先，但不要做成纯展板
- 顶部信息区明显放大
- 卡片适度变大
- 每屏尽量保持 `4` 个左右主卡片

### 2.3 首屏目标

用户已确认训练页需要满足：

- 首屏尽量完整看到主要操作区和 `4` 张卡片
- 不依赖用户先下滚才能进入训练主动作

## 3. 设计原则

### 3.1 大屏优先，但兼顾普通窗口可用

本轮不追求“所有窗口下完全同形”，而是采用以下策略：

- 在 `27` 寸触摸屏上，页面应明显更舒展、更稳定
- 在普通浏览器窗口中，页面仍需保持可读和可点击
- 不为了兼容小窗而牺牲大屏主体验

### 3.2 儿童侧界面优先强调动作，而不是信息

儿童训练主路径的界面应当先让人一眼看到：

- 这里是什么模式
- 现在该点哪里
- 哪些卡片是当前主要操作对象

因此页面结构应优先放大：

- 模式标题
- 主操作按钮
- 当前状态提示
- 主卡片区

而不是继续堆叠说明性文案。

### 3.3 触摸目标必须比当前更大

所有关键交互元素都应比当前版本更适合手指点击，包括：

- 首页入口卡
- 类别切换按钮
- 返回主页按钮
- 引导训练的“重新播放声音”
- 引导训练候选卡

### 3.4 首屏必须有明确的视觉节奏

训练页首屏结构统一为：

1. 页面标题区
2. 操作控制区
3. 状态反馈区
4. 主卡片区

这样可以降低扫视负担，避免用户把视线在说明文案和卡片区之间来回跳。

## 4. 页面级设计

### 4.1 首页

#### 4.1.1 布局目标

首页应从“中间一小块面板 + 两个还不够大的按钮”调整为：

- 更宽的主面板
- 更强的标题区
- 更有重量的双入口卡布局

#### 4.1.2 结构调整

首页仍保留当前信息结构，不新增模块，只调整布局尺度：

- 主面板宽度明显放宽
- 标题区纵向占比增加
- 两个训练入口卡占据首屏视觉主角位置

#### 4.1.3 尺寸方向

建议实现口径：

- `.screen-panel` 在首页允许扩展到约 `1360px ~ 1480px`
- 首页标题区最大可读宽度提升到约 `720px ~ 880px`
- 两个主入口卡维持 `2` 列，但单卡最小高度提升到约 `300px ~ 340px`
- 卡片标题字号比当前再上一个层级
- 卡片描述宽度适度放宽，避免文字局促

#### 4.1.4 视觉结果

首页首屏应达到以下效果：

- 视觉重心明确落在两个训练入口上
- 大屏下不再显得空旷又局促
- 一眼知道应用的两个主入口，没有“说明太多、动作太轻”的问题

### 4.2 自由探索

#### 4.2.1 布局目标

自由探索页需要从“说明 + 分类 + 状态 + 卡片”的普通页面，调整为适合大屏训练的稳定首屏：

- 标题区更强
- 分类区更像触摸控制排
- 状态区更独立
- 主卡片区首屏尽量稳定显示 `4` 张主卡

#### 4.2.2 结构调整

页面仍保留现有内容顺序，但调整为更强层次：

1. 标题区与返回按钮形成稳定页头
2. 分类按钮区整体放大
3. 播放状态区独立成带状区块
4. 素材卡区在大屏下默认优先形成 `4` 列

#### 4.2.3 尺寸方向

建议实现口径：

- 页面主面板宽度提升到约 `1440px ~ 1560px`
- 页头标题宽度增大，返回按钮高度提升到约 `64px ~ 72px`
- 分类按钮最小高度提升到约 `64px ~ 72px`
- 状态区从单行文本提升为有内边距的独立条带
- 卡片网格在大屏下优先使用 `4` 列
- 单卡最小高度提升到约 `300px ~ 340px`
- 图片展示区保持方形，但整体面积放大

#### 4.2.4 响应式口径

自由探索页建议采用分段布局：

- 大屏：`4` 列
- 中屏：`3` 列
- 较窄窗口：`2` 列
- 手机级宽度：`1` 列

注意：大屏布局不再依赖 `auto-fit + 220px` 这类偏保守的自适应策略，而应优先保证首屏训练节奏。

#### 4.2.5 视觉结果

自由探索页首屏应达到：

- 用户不滚动也能立即进入点击训练
- 分类按钮不再显小
- “正在播放 / 当前类别”更容易被看到
- `4` 张主卡在大屏下分布均匀，不挤、不散

### 4.3 引导训练

#### 4.3.1 布局目标

引导训练页需要比自由探索页更强调“先听，再选”，因此首屏应突出：

- 当前训练任务
- 重播声音入口
- 回答反馈
- 候选卡片

#### 4.3.2 结构调整

页面仍保留当前功能模块，但改成更清晰的首屏节奏：

1. 页头区：模式标题 + 返回按钮
2. 控制区：题目提示 + 重播按钮
3. 反馈区：回答状态与边界提示
4. 候选区：主卡片网格
5. 通过后再显示下一题按钮

#### 4.3.3 尺寸方向

建议实现口径：

- 页面主面板宽度与自由探索保持同级
- “请找出刚才播放的声音”所在控制带明显增高并增强内边距
- `重新播放声音` 按钮高度与返回按钮保持统一量级
- 反馈区独立成条带，不再只是轻量一行字
- 候选卡在大屏下默认按 `4` 张主卡布局

#### 4.3.4 与候选数量的关系

当前教师设置允许 `2 / 3 / 4` 个候选项，因此布局必须兼容：

- 当候选数为 `4` 时：使用标准 `4` 列布局
- 当候选数为 `3` 时：保持 `3` 张卡横向拉开，避免一侧留大空洞
- 当候选数为 `2` 时：让两张卡明显变宽，仍保持首屏强视觉重心

这意味着引导训练页不应只依赖统一的 `auto-fit` 网格，而应允许根据候选项数量切换网格列数或卡片跨度。

#### 4.3.5 视觉结果

引导训练页首屏应达到：

- 一眼知道先听声音、再点图片
- `重新播放声音` 足够显眼，不会埋在页面里
- 正确 / 请再试一次的反馈更容易被捕捉
- 候选卡既大又稳，不会在大屏上显得零散

## 5. 全局样式方向

### 5.1 主容器

本轮需要调整的核心不是“换一套新皮肤”，而是放宽儿童侧主容器：

- `.app-shell` 在大屏下减少“过度居中缩小”的观感
- `.screen-panel` 在儿童侧页面允许使用更宽的最大宽度
- 保留现有奶油底色、圆角和阴影体系，不重新设计配色

### 5.2 字号与间距

本轮应整体提升：

- 页面标题字号
- 状态文案字号
- 卡片标题字号
- 关键区块的上下内边距

但不应把说明文案堆得更长。

### 5.3 按钮体系

本轮应统一抬高以下元素的触摸尺寸：

- `.primary-tile`
- `.secondary-action`
- `.category-chip`
- `.guided-replay`
- `.guided-next`

原则是：重要按钮的高度与大屏触摸动作匹配，而不是仅满足鼠标点击。

## 6. 代码改动边界

### 6.1 允许改动

本轮允许改动的重点文件包括：

- `src/styles/global.css`
- `src/screens/home-screen.tsx`
- `src/modules/explore/explore-screen.tsx`
- `src/modules/explore/explore-card.tsx`
- `src/modules/guided/guided-screen.tsx`

必要时可补充极少量结构类名，以支撑布局实现。

### 6.2 尽量不动

以下文件原则上不应因本轮布局调整而改动逻辑：

- `src/modules/explore/use-explore-session.ts`
- `src/modules/guided/use-guided-session.ts`
- `src/assets/manifest/builtin-assets.json`
- `public/builtin/**/*`

### 6.3 禁止影响的行为

本轮实现后必须保持：

- `44` 个正式内置素材仍然全部可见
- 所有正式素材仍然可播放对应声音
- 自由探索的分类切换逻辑不变
- 引导训练的题目生成和答题记录逻辑不变

## 7. 验收标准

本轮验收以以下结果为准：

### 7.1 首页

- 大屏下首页主面板明显更宽
- 两个训练入口卡明显更大
- 首页首屏不再显得偏小、偏空

### 7.2 自由探索

- 首屏可见完整标题区、分类区、状态区和 `4` 张主卡
- 分类按钮和返回按钮触摸面积明显提升
- 卡片图片区和标题区比当前版本更舒展

### 7.3 引导训练

- 首屏可见标题区、控制区、反馈区和主候选卡
- `重新播放声音` 更显眼
- 候选项在 `2 / 3 / 4` 个时都保持平衡布局

### 7.4 功能回归

- 不出现素材丢失
- 不出现图片不显示
- 不出现声音不播放
- 不出现训练主流程失效

## 8. 验证方式

实现完成后至少需要提供：

- 首页新布局截图
- 自由探索新布局截图
- 引导训练新布局截图
- 与当前 `44` 项正式内置素材显示 / 播放未受影响相关的验证证据

## 9. 风险与控制

### 9.1 风险

- 过度放宽容器后，普通窗口下可能出现拥挤或折行异常
- 仅靠 CSS 网格放大，可能无法兼顾引导训练 `2 / 3 / 4` 候选项的平衡布局
- 若误改逻辑层，可能引入素材显示或播放回归

### 9.2 控制措施

- 使用分段响应式规则，而不是只拉大单一最大宽度
- 对引导训练候选区增加与候选数量相关的布局控制
- 将改动重点限制在布局类名和样式层
- 通过现有测试和手动验证确认素材显示 / 播放不回归
