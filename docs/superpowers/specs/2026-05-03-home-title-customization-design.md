# 首页标题与软件名称自定义设计说明

## 1. 目标与范围

### 1.1 本轮目标

在不影响现有训练逻辑、素材数据、播放能力和统计逻辑的前提下，完成以下改动：

- 首页将 `特殊儿童声音感知训练` 调整为居中的主标题
- 首页主标题字号明显放大
- 去掉首页提示文案 `请选择今天的训练方式`
- 在教师管理台新增一个可保存的 `软件应用标题` 设置
- 自定义标题保存后，同时影响：
  - 首页主标题
  - 浏览器页标题 `document.title`
  - Tauri 窗口标题

### 1.2 本轮范围

本轮只涉及以下区域：

- `首页`
- `教师管理台 > 训练设置`
- `设置存储层`
- `应用路由层的标题同步逻辑`

### 1.3 明确不做

以下内容不进入本轮：

- 不调整训练入口按钮文案
- 不调整自由探索或引导训练页面标题
- 不修改训练逻辑、出题逻辑、统计逻辑
- 不修改素材清单、图片、音频和播放链路
- 不回头处理桌面环境、白屏或素材接入问题

## 2. 已确认决策

### 2.1 标题策略

采用单一 `软件应用标题` 设置，统一驱动以下三个位置：

- 首页主标题
- 浏览器页标题
- Tauri 窗口标题

不拆分成多个独立字段，避免教师台出现多份标题设置后产生不一致。

### 2.2 默认值与回退

默认标题保持为：

- `特殊儿童声音感知训练`

如果教师输入为空字符串或仅空白字符，运行时统一回退到默认标题，不保存一个“视觉上为空”的应用标题。

## 3. 设计原则

### 3.1 首页只保留一个核心标题

首页首屏已经有两个主训练入口卡，不需要额外的提示句。视觉层级应收敛为：

1. 软件名称
2. 两个训练入口
3. 次级操作（教师入口、退出）

因此本轮将首页顶部改为单一大标题，不再保留“请选择今天的训练方式”这一层。

### 3.2 标题修改必须即时影响应用壳层

教师修改标题后，不能只改首页文本而遗漏页签或窗口壳。对用户来说，软件名称是一个统一概念，因此保存后应同步影响：

- React 页面中的首页标题
- 浏览器文档标题
- Tauri 窗口标题

### 3.3 兼容已有数据库快照

当前应用已经存在实际持久化数据库，新增 `app_title` 不能只修改 `schema.ts`。必须补运行时迁移，确保老数据库也能自动补齐新列并拥有默认值。

## 4. 数据与存储设计

### 4.1 设置模型扩展

`TrainingSettings` 增加字段：

- `appTitle: string`

其职责是保存教师配置的软件名称。

### 4.2 数据库字段

`settings` 表新增列：

- `app_title text not null default '特殊儿童声音感知训练'`

### 4.3 迁移策略

由于 `create table if not exists` 不会修改已有表结构，本轮需要在运行时加一个轻量迁移步骤：

- 启动数据库后检查 `settings` 表列信息
- 如果缺少 `app_title`，执行 `alter table settings add column app_title text not null default '特殊儿童声音感知训练'`

迁移应早于 `ensureDefaultSettings` 与设置读取，否则老库会在加载设置时直接报错。

### 4.4 默认设置初始化

`ensureDefaultSettings` 的默认插入语句同步写入：

- `app_title = '特殊儿童声音感知训练'`

这样新库和老库迁移后的行为一致。

## 5. 页面与交互设计

### 5.1 首页

首页结构调整为：

- 继续保留 `hero-copy` 区块
- 去掉当前副标题式眉标写法
- 将软件标题提升为首页唯一主标题
- 主标题水平居中
- 主标题字号明显高于当前版本

视觉结果应是：

- 首屏最先看到的是软件名称
- 训练入口卡仍然是第二视觉焦点
- 页面不再出现“标题像眉标、真正 H1 却是提示句”的层级错位

### 5.2 教师管理台

教师台新增一个文本输入项：

- 标签：`软件应用标题`

交互规则：

- 加载时显示当前保存值
- 编辑后点击 `保存设置` 才正式生效
- 输入会被 `trim`
- 若 `trim` 后为空，则回退成默认标题再保存

位置上将它放入现有训练设置表单，与柔和模式、候选项数量、默认全屏同属一组设置，不单独开新模块。

## 6. 运行时同步设计

### 6.1 标题状态归属

`AppRouter` 负责持有当前应用级设置状态，至少包括：

- `appTitle`
- `guidedOptionCount`

原因：

- 路由层已经负责把教师设置传给引导训练
- 软件标题属于应用壳级别，不应只停留在首页组件里

### 6.2 设置刷新时机

需要有统一的 `refreshTrainingSettings` 行为，用于从当前 `settingsStore` 重新读取设置，并同步更新：

- `appTitle`
- `guidedOptionCount`

触发时机：

- 路由初始化后首次加载
- 进入引导训练前
- 教师台保存成功后

这样可以保证标题和候选项数量都基于同一份最新设置，而不是分别走不同的读取路径。

### 6.3 浏览器标题同步

当 `appTitle` 状态变化时：

- 设置 `document.title = appTitle`

这会覆盖当前 `index.html` 中的临时标题 `tmp-vite`。

### 6.4 Tauri 窗口标题同步

当运行在 Tauri 环境时，`appTitle` 状态变化后同步调用窗口标题更新：

- `getCurrentWindow().setTitle(appTitle)`

如果当前不在 Tauri 环境，跳过该步骤，不影响浏览器预览与测试。

说明：

- `tauri.conf.json` 里的窗口标题仍保留默认值，作为冷启动前的基础配置
- React 运行起来后，再由运行时设置覆盖为教师保存的标题

## 7. 测试设计

### 7.1 设置存储测试

需要补充 `settings-store` 测试，验证：

- 默认设置包含 `appTitle`
- 保存自定义标题后再次读取能拿到该值
- 空白输入会在上层规范化后回退为默认标题

### 7.2 教师台测试

需要补充 `settings-screen` 测试，验证：

- 页面出现 `软件应用标题` 输入框
- 能正确展示已加载的标题
- 修改后点击保存，会将更新后的 `appTitle` 传给 `saveTrainingSettings`

### 7.3 首页测试

需要补充 `home-screen` 测试，验证：

- 首页显示传入的软件标题
- 不再渲染 `请选择今天的训练方式`

### 7.4 路由集成测试

需要补充 `AppRouter` 级别测试，验证完整用户路径：

- 进入教师台
- 修改 `软件应用标题`
- 点击保存
- 返回主页后看到新的首页主标题
- `document.title` 同步更新为新标题

## 8. 风险与控制

### 8.1 老数据库缺列

风险：

- 已存在数据库快照没有 `app_title`

控制：

- 在数据库初始化链路中添加显式迁移，不依赖 `create table if not exists`

### 8.2 浏览器与 Tauri 标题不同步

风险：

- 只更新 React 视图，没有更新壳层标题

控制：

- 将标题同步逻辑放到路由层统一处理，不把它分散到首页组件与教师组件各自维护

### 8.3 空标题导致首页显示异常

风险：

- 教师输入空白后，首页和页签出现空标题

控制：

- 保存前统一 `trim`
- 空值回退到默认标题

## 9. 验收标准

本轮完成后，以下行为必须成立：

- 首页顶部只显示一个居中的大标题
- 首页不再显示 `请选择今天的训练方式`
- 教师台可以修改并保存 `软件应用标题`
- 保存后返回主页，首页标题立即变成新值
- 浏览器页签标题同步变成新值
- Tauri 桌面窗口标题同步变成新值
- 不影响现有训练入口、素材显示、声音播放与统计逻辑
